Portability & command line

While CC is meant to be a fully portable application, there are a few features that don't work from a portable app. Especially registered file extensions and COM support must be registered on the local machine.
CC requires the .NET framework on it's target machine!

Portability

As mentioned above, CC is meant to be portable. It doesn't store any data in the registry nor in your AppData folder (Note: for now it does. This will be removed later). So you can take CC on your USB-Stick with you, as well as you can "uninstall" it by just deleting its directory. Don't forget to backup your resource files before!

CC is neither packaged in the PortableApps format, nor is it likely to become packaged. CC is written in C#, so it requires the .NET framework. PortableApps does not accept this.

To keep portability while offering comfortable features such as registered file extensions, CC supports some command line switches.


Command line

  • --install_ext causes CC to install the *.ccp and *.ccr file extensions and to register itself as default handler. If run on a machine which has already installed these, CC will instead uninstall them.
    The app is shutdown without opening the main window.

  • --install_COM causes CC to install its COM features to the local machine. Running it a second time uninstalls them.
    The app is shutdown without opening the main window.

  • --install_full installs or uninstalls both the COM features and the file extensions.
    The app is shutdown without opening the main window.

  • If you pass the name of a file to CC, it will try to read it as *.ccr or *.ccp file and parse the contained resources.

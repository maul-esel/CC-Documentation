Developers

This section contains tips, information and guidelines for developers who either want to participate in developing CC or want to create their own plugin.
Before reading this, make sure you have read and understood the concepts. This is required for CC developers as well as for plugin developers.
And last but not least don't forget to read the topic on Resource syntax.

Developing ChameleonCoder

The easiest way to participate in CC development is to go for github and create a new fork. Before changing anything to the concepts you should contact me. It's no problem, I change them a lot, but I need to know. To develop CC, you should use C#.


Developing plugins

You can develop plugins in any .NET language. Well, not any: you currently cannot develop plugins in dynamic languages like IronPython or IronRuby. However, if there's enough demand for it I will consider adding this at a later point.
For more information on the plugin types you can create, see Plugin Types. More information on developing plugins in general can be found here.
